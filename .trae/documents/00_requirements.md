# 专属双人私密 IM（网页）需求说明

## 目标
- 无账号体系：不注册、不登录。
- 名字 + 对方名字 + 暗号建立唯一双人通道。
- 支持文本 / 图片 / 视频消息；消息在聊天内直接预览。
- 通过 Socket.IO 实时同步，服务端可持久化存储。

## 范围（本期实现）
### P0（必须）
- 通道建立页
  - 输入：自身名字、对方名字、暗号（暗号 2~20 位、可显示/隐藏）。
  - 实时校验：字段为空或暗号不合法时，“建立通道”置灰。
  - 匹配：双方输入的（自身名字、对方名字、暗号）互为对应且一致时进入同一通道。
  - 失败提示：统一提示“匹配失败，请检查名字或暗号是否正确”，不暴露具体项。
  - 支持返回并重新匹配。

- 聊天页
  - 文本消息：Enter 发送；Ctrl+Enter 换行。
  - 图片消息：单张上传（≤50MB），聊天内缩略图，点击全屏预览。
  - 视频消息：单个上传（≤50MB），聊天内封面缩略图，点击全屏播放；预览默认静音，可切换。
  - 发送状态：发送中、成功、失败（可重试）。
  - 历史消息：进入通道加载历史；PC/H5 同步一致。
  - 清空聊天记录：需二次确认；清空后双端同步清空。
  - 断线提示：服务端不可用时提示“服务器连接失败，请稍后重试”，进入只读模式（仅查看本地缓存与已加载历史）。

### P1（后续）
- 深色模式。
- 预览页支持保存到本地。

### P2（后续）
- PC 自定义背景色。

## 核心规则
### 通道唯一性
- 相同的（名字组合 + 暗号）映射到同一条会话通道；再次输入可恢复会话。

### 双向匹配
- A 输入：自身=A，对方=B，暗号=K
- B 输入：自身=B，对方=A，暗号=K
- 两端应进入同一通道。

## 非功能需求
- 适配：PC（Chrome/Edge/Firefox/Safari 主流版本）与 H5（iOS/Android 主流浏览器）。
- 极简无干扰：无广告、无推荐、无好友列表、无搜索。
- 开源部署：Node.js 一键安装依赖与启动；部署步骤≤5 步（文档提供）。

## 数据与隐私约束（实现原则）
- 不引入账号体系。
- 暗号不应在 UI 或日志中明文展示；仅用于通道验证与派生会话标识。
- 媒体文件访问路径可配置（默认本地文件系统）。
